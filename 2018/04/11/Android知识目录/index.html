多线程设计模式

不用锁保证线程安全

  • 不可变对象

    • CopyOnWriteArrayList
    • 直接替换要操作的对象，以保证线程安全性

  • 线程特有存储

    • ThreadLocalRandom
  • 串行线程封闭
    • Serializer
    • TaskProcessor
    • WorkerThread
    • 不使用锁保证线程安全。使用单个线程与队列处理任务；本质上使用维护队列的小锁代替维护多线程安全的大锁

停止线程

  • 两阶段终止
    • ThreadPoolExecutor
    • 调用shutdown后，工作队列的真正停止分为准备停止与实际停止两个阶段

线程协作

  • 保护性暂挂
    • LinkedBlockingQueue
    • 直到执行完某操作 ( 有数据 ) 时才继续任务
  • 生产者消费者
    • ThreadPoolExecutor
    • 内部维护了工作队列与工作线程，外部客户端相当于生产者，产品是Runnable对象

提高并行性、减少等待

  • 承诺
    • Promisor ( Callable )
    • Promise ( Future )
    • TaskExecutor ( ExecutorService )
  • 主动对象
    • Scheduler ( ExecutorService )
    • ActivationQueue ( BlockingQueue )
    • MethodRequest ( Callable )
    • Future ( Future )
    • 讲任务的提交与任务的执行策略分离
  • 流水线
    • 讲一个任务处理分解成若干个处理阶段，每个阶段是顺序执行的，都有相应的工作者线程或线程池执行计算，各个阶段可以是并行的

提高响应性

  • 主仆
    • Master
    • Slave
    • TaskDivideStrategy
    • TaskDispatchStrategy
    • ResultRepository
    • RetryPollcy
    • 基于分而治之的思想，将大任务分解为若干子任务
  • 半同步/半异步
    • 同步层
    • 队列层
    • 异步层
    • 将任务分为同步层和异步层

减少资源消耗

  • 线程池
    • 工作队列的类型
      • 有界队列
      • 无界队列
      • 直接交接队列
    • 工作任务的类型
